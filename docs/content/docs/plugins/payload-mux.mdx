`@pro-laico/payload-mux` brings [Mux Video](https://www.mux.com/) to Payload. It adds a
**Videos** collection (or extends one you name) whose admin field uploads directly to Mux,
exposes virtual playback / poster / gif URLs, and stays in sync with Mux through a webhook â€”
delete a video in Payload and its Mux asset goes too, and vice versa.

```bash
pnpm add @pro-laico/payload-mux @mux/mux-player-react
```

<Callout type="info">
  Ported and maintained from [`@oversightstudio/mux-video`](https://github.com/oversightstudio/payload-plugins/tree/main/packages/mux-video)
  (MIT) â€” kept as a first-party package to track Payload and Mux updates directly.
</Callout>

## Setup

Set your Mux credentials as the standard `MUX_*` environment variables â€” the plugin and the
Mux SDK read them automatically, so adding the plugin takes no arguments. Then point a Mux
webhook at `/api/mux/webhook` (or `<your routes.api>/mux/webhook` if you've customized
Payload's API route).

```bash
MUX_TOKEN_ID=...
MUX_TOKEN_SECRET=...
MUX_WEBHOOK_SECRET=...        # signed playback also: MUX_SIGNING_KEY, MUX_PRIVATE_KEY
```

## The Videos collection

The plugin registers a `mux-video` collection. Mux fills in the read-only metadata; you set
`title` and an optional `posterTimestamp`.

| Field | Type | Notes |
| --- | --- | --- |
| `muxUploader` | ui | The admin uploader + player. |
| `title` | text | Unique, required. |
| `assetId` | text | Mux asset id (read-only). |
| `duration` | number | Read-only. |
| `posterTimestamp` | number | Seconds into the video for the poster; defaults to the middle. |
| `aspectRatio` / `maxWidth` / `maxHeight` | â€” | Read-only, from Mux. |
| `playbackOptions[]` | array | Per playback id: `playbackId`, `playbackPolicy`, and virtual `playbackUrl` / `posterUrl` / `gifUrl`. |

## Use a video

Relate to `mux-video` from any collection, then play it back with `@mux/mux-player-react`:

```tsx
import config from '@payload-config'
import MuxPlayer from '@mux/mux-player-react'
import { getPayload } from 'payload'

export default async function Page() {
  const payload = await getPayload({ config })
  const video = await payload.findByID({ collection: 'mux-video', id: 'example' })
  const playback = video.playbackOptions![0]

  return <MuxPlayer src={playback.playbackUrl!} poster={playback.posterUrl!} />
}
```

## Endpoints

The plugin registers three endpoints under Payload's API route:

- `POST /api/mux/upload` â€” mint a Mux direct-upload (the admin uploader posts files to it).
- `GET /api/mux/upload?id=â€¦` â€” read a direct-upload, to pick up its `asset_id`.
- `POST /api/mux/webhook` â€” verify and apply Mux events: set metadata on ready/updated,
  delete on deleted, and (with `autoCreateOnWebhook`) backfill assets uploaded in Mux.

## Seeding from local files

Local-file seeding lives in the **seed plugin**, not here â€” the two packages stay fully
decoupled (neither imports the other). [`@pro-laico/payload-seed`](/docs/plugins/payload-seed)
exposes a Mux helper on its `./mux` subpath that uploads **local video files** to Mux exactly
as the admin uploader would (direct upload â†’ poll until ready â†’ create the `mux-video` doc).
It reads this plugin's credentials from `config.custom` by convention, so you pass only
`payload` and the videos:

```ts
// scripts/seed-mux.ts â€” run with `payload run scripts/seed-mux.ts`
import { seedMuxVideos } from '@pro-laico/payload-seed/mux'
import { getPayload } from 'payload'
import config from '../src/payload.config'

const payload = await getPayload({ config })
await seedMuxVideos(payload, {
  dir: 'seed-assets',
  clear: 'tagged', // idempotent reseed; or 'all' to wipe the whole (dev) Mux environment
  videos: [{ title: 'Intro', file: 'intro.mp4', playbackPolicy: 'public' }],
})
```

<Callout type="info">
  This plugin's part is passive: the `mux-video` `beforeChange` hook detects pre-resolved data
  (an `assetId` + `playbackOptions` already present) and skips the redundant Mux round-trip, so
  seeded docs are created cleanly. To use the helper, install `@mux/mux-node` (an optional peer
  of the seed plugin). See the [seed plugin docs](/docs/plugins/payload-seed#mux-video-seeding).
</Callout>
