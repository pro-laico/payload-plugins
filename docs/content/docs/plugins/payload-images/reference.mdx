`imagesPlugin(options?)` is the single entry point. One call registers the collections, the
transform and purge endpoints, and the admin UI. It's zero-config — everything below is a knob you
can turn later.

    defaultQuality: 75,
    qualityRange: [40, 95],
    defaultFormat: 'auto',

## Environment variables

Two env vars override the transform endpoint's concurrency knobs (handy for serverless, where you
tune per-process throughput without changing code).

| Variable | Default | What |
| --- | --- | --- |
| `IMAGES_TRANSFORM_CONCURRENCY` | `cpus - 1` (computed) | Max concurrent Sharp transforms in this process. Overrides `transform.maxConcurrency`. |
| `IMAGES_SHARP_CONCURRENCY` | `1` | Per-image libvips thread cap; `0` = CPU cores. Overrides `transform.sharpConcurrency`. |

## CLI commands

The plugin registers two Payload CLI commands (run via `pnpm payload <command>`).

| Command | Flags | What |
| --- | --- | --- |
| `payload images:backfill` | `--force`, `--focal`, `--collection` | Stamps upload-time metadata (placeholder tiers, palette, alpha flags, saliency focal point) onto images that predate the plugin. See [Metadata → Backfilling an existing library](/docs/plugins/payload-images/metadata#backfilling-an-existing-library). |
| `payload images:prewarm` | `--now`, `--limit` | Bulk-warms a library against the learned render profiles. Requires `prewarm` on. See [Caching → Smart prewarm](/docs/plugins/payload-images/caching#smart-prewarm). |

## Endpoints

| Route | What |
| --- | --- |
| `GET /api/img/:id` | The on-demand transform endpoint: reads the image `id` and the transform query params, renders the variant once, then caches it. See [Image URLs](/docs/plugins/payload-images/image-urls). |
| `POST /api/img/purge/:id` | Purge this image's cached variants (the **Preset manager** panel's purge button posts here). |

## Collections & globals

The plugin registers `images` (the upload collection you touch), the hidden `generated-images`
variant cache, and — with `prewarm` on — the hidden `image-render-profiles` registry, all under an
**Assets** admin group. See [Collections](/docs/plugins/payload-images/collections) for their
fields, hooks, and access.
